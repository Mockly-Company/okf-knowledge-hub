# 화면 구성

화면 구성은 아직 확정하지 않았습니다. 이 문서는 합의된 기능을 기준으로 필요한 화면과 책임을 정리하며, 최종 좌측 탐색 구조는 유스케이스와 정보 모델이 승인된 뒤 결정합니다.

## 앱 수준 화면

### 워크스페이스 시작 화면

- 최근 워크스페이스
- 워크스페이스 만들기
- Git 저장소에서 가져오기
- GitHub 로그인 상태

### 워크스페이스 설정

- OKF 지식 저장소
- 백엔드·프론트엔드 코드 저장소
- GitHub 인증과 동기화
- 기본 브랜치
- 제품 구조와 문서 템플릿
- Codex/MCP 연결 안내

프로젝트 저장소 연결은 워크스페이스의 좌측 주 메뉴에 두지 않습니다.

## 워크스페이스 화면 후보

### Home

워크스페이스에 들어왔을 때 무엇을 먼저 처리해야 하는지 보여줍니다.

- 최근 활동
- 진행 중인 사용자 시나리오
- 응답 대기와 막힘
- 내가 검토할 요청
- 최근 완료와 배포

### Work

- Kanban Board
- GitHub Issues
- 내 작업
- 진행 상태와 필터

Board와 Issues를 하나의 Work 영역 아래에 둘지 별도 화면으로 둘지는 미결정입니다.

### Knowledge

- 제품 구조 탐색
- 문서 라이브러리
- 문서 편집과 렌더링
- 문서별 Git history
- 관련 분석·테스트·API·코드
- 현재 버전과 제안 버전 전환

Knowledge 화면에서는 필요할 때만 제품 구조 트리를 열고, 평소에는 최근 문서·현재 작업·연결된 자료를 중심으로 이동하는 방향을 검토합니다.

### Reviews

- 내가 받은 요청
- 내가 보낸 요청
- 선택형 결정 요청
- 완료된 리뷰
- 현재 문서와 제안된 문서 비교
- Mermaid 이전/제안 렌더링 비교
- 댓글, 승인과 수정 요청

### Activity

- 날짜별 설계·리뷰·결정·구현·테스트·배포 이력
- 사용자, 기능과 문서 종류 필터
- 관련 문서, Issue, PR과 커밋으로 이동

### API Catalog

OpenAPI를 원본으로 사용할 경우 별도 API 탐색 화면을 제공합니다.

- 서비스 또는 태그별 API 목록
- Path, method와 `operationId` 검색
- 요청·응답 스키마와 오류 계약
- 관련 기능과 처리 케이스
- 변경 제안과 버전 diff

API를 제품 구조 트리에 직접 중첩하지 않습니다.

## 공통 작업 화면

문서, 코드와 리뷰를 동시에 다루는 화면은 다음 영역을 조합할 수 있습니다.

```text
현재 문맥 탐색 | 문서 또는 리뷰 본문 | 관련 코드·API·변경 정보
```

- 좌측 영역은 항상 고정된 전체 트리가 아니라 현재 작업에 필요한 탐색을 제공합니다.
- 우측 보조 영역은 관련 코드, diff, 댓글 또는 속성으로 전환됩니다.
- 문서와 코드는 같은 화면에서 읽되 코드를 직접 편집하지 않습니다.

## 현재 UI 기술 후보

### 합의된 방향

- Tauri + 웹 프론트엔드
- shadcn/ui 기반 컴포넌트
- 일반 화면은 여백이 있는 디자인, 트리·코드·테이블은 compact 밀도
- 문서 편집은 MDXEditor 계열 리치 에디터
- Markdown 원문과 diff는 CodeMirror

이전에 사용한 `Nova`는 디자인 스타일 후보 이름이며 프로젝트명이 아닙니다.

### 미결정

- 전역 좌측 메뉴와 상황별 트리의 조합
- Home 외 화면에서 상단 탭을 사용하는 범위
- 제품 구조 트리를 항상 표시할지 필요할 때만 표시할지
- 리뷰 목록과 상세 화면의 분할 방식
