# 화면 구성

이 문서는 합의된 전역 화면 구조와 아직 구체화 중인 화면별 책임을 정리합니다.

## 앱 수준 화면

### 워크스페이스 시작 화면

- 최근 워크스페이스
- 워크스페이스 만들기
- Git 저장소에서 가져오기
- GitHub 로그인 상태

### 워크스페이스 설정

- OKF 지식 저장소
- 백엔드·프론트엔드 코드 저장소
- GitHub 인증과 동기화
- 기본 브랜치
- 제품 구조와 문서 템플릿
- Codex/MCP 연결 안내

프로젝트 저장소 연결은 워크스페이스의 좌측 주 메뉴에 두지 않습니다.

## 워크스페이스 전역 구조

**상태: 합의**

```text
Home
Knowledge
Issues
────────
Settings
```

- `Work`는 독자적인 데이터와 행동이 없어 전역 메뉴에서 제외합니다.
- `Activity`는 Home의 타임라인으로 흡수합니다.
- `Reviews`는 독립 콘텐츠가 아니라 문서나 PR에 붙는 처리 과정이므로 전역 메뉴에서 제외합니다.
- 사용자가 처리해야 할 리뷰와 결정 요청은 Home에서 모아 보여주고, 실제 검토는 원래 문서나 PR 문맥에서 진행합니다.

### Home

워크스페이스에 들어왔을 때 무엇을 먼저 처리해야 하는지 보여줍니다.

- 최근 활동
- 전체 진행 상황과 막힌 작업
- 내가 검토할 요청
- 최근 완료와 배포
- 날짜별 활동 타임라인

### Knowledge

- 제품 구조 탐색
- 문서 라이브러리
- 문서 편집과 렌더링
- 문서별 Git history
- 관련 분석·테스트·API·코드
- 현재 버전과 제안 버전 전환

Knowledge는 문서가 중심인 공간입니다. 진행 중인 작업을 별도 목록으로 소유하지 않으며, 문서에서 관련 Issue나 리뷰 상태를 참조 정보로 확인할 수 있습니다.

### Issues

- GitHub Issue Board
- 전체 Issue 목록
- 나에게 할당된 Issue
- Milestone과 필터
- Issue 생성, 수정, 댓글과 종료
- 관련 기능·시나리오·처리 케이스·문서 표시
- 연결된 PR과 코드 확인

Board의 카드 원본은 GitHub Issue로 둡니다. 사용자 시나리오 전체 진행률은 Home에서 관련 Knowledge 상태와 Issue를 조합해 보여줍니다.

### 문맥 안의 리뷰

- Knowledge의 변경 제안에서 문서·Mermaid diff와 댓글을 확인합니다.
- Issue 또는 PR에서 코드 diff, 댓글, 승인과 수정 요청을 처리합니다.
- 선택형 결정은 대상 문서나 PR의 맥락 안에서 처리합니다.
- Home의 확인할 요청에서 해당 문서나 PR로 이동합니다.

### API Catalog

OpenAPI를 원본으로 사용할 경우 Knowledge 안에 API 탐색 화면을 제공합니다.

- 서비스 또는 태그별 API 목록
- Path, method와 `operationId` 검색
- 요청·응답 스키마와 오류 계약
- 관련 기능과 처리 케이스
- 변경 제안과 버전 diff

API를 제품 구조 트리에 직접 중첩하지 않습니다.

## 공통 작업 화면

문서, 코드와 리뷰를 동시에 다루는 화면은 다음 영역을 조합할 수 있습니다.

```text
현재 문맥 탐색 | 문서 또는 리뷰 본문 | 관련 코드·API·변경 정보
```

- 좌측 영역은 항상 고정된 전체 트리가 아니라 현재 작업에 필요한 탐색을 제공합니다.
- 우측 보조 영역은 관련 코드, diff, 댓글 또는 속성으로 전환됩니다.
- 문서와 코드는 같은 화면에서 읽되 코드를 직접 편집하지 않습니다.

## 현재 UI 기술 후보

### 합의된 방향

- Tauri + 웹 프론트엔드
- shadcn/ui 기반 컴포넌트
- 일반 화면은 여백이 있는 디자인, 트리·코드·테이블은 compact 밀도
- 문서 편집은 MDXEditor 계열 리치 에디터
- Markdown 원문과 diff는 CodeMirror

이전에 사용한 `Nova`는 디자인 스타일 후보 이름이며 프로젝트명이 아닙니다.

### 미결정

- Home의 진행 상황·확인할 요청·타임라인 배치
- Knowledge의 Documents·Structure·History·APIs 하위 탐색 방식
- 제품 구조 트리를 항상 표시할지 필요할 때만 표시할지
- Home의 요청 목록에서 문서·PR 검토 화면으로 전환하는 방식
- Issues의 Board·List·My Issues·Milestones 하위 탐색 방식
